# Methodology rebuild — jury-report-based version

This package contains two replacement files:

- `methodology-data.js`: 20 progressive modules, entirely in English, with no universal minute-based rules or diagnostic questions about minutes.
- `methodology.html`: the current page design, updated for 20 modules, English accessibility labels, and the relocated Context Builder, Key Question Workshop and Full Colle Simulator.

## Installation

1. In the GitHub repository, replace the existing `methodology-data.js` with the new file.
2. Replace the existing `methodology.html` with the new file.
3. Commit both files in the same commit.
4. Wait for GitHub Pages to deploy, then reload `methodology.html` with Ctrl + F5.

## What changed

- 20 modules based on recurrent jury expectations and warnings.
- No universal timing chart or questions asking for a number of minutes.
- Explicit exam-format checking before any method is applied.
- Separate modules for audio, video, speaking notes, restitution/commentary, real issue, civilisation use, delivery, interview and feedback.
- Every module includes jury expectations, common mistakes, a step-by-step method, a worked example, weak/effective comparison, guided exercises, an oral task, classroom/colle transfer and a self-check.
- The classroom and regular colles remain the real place where skills are corrected and consolidated.
